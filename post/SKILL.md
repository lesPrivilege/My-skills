# 帖子分析工作流

用戶貼了一段文字（社區帖子、文章、討論串、推文串等內容），按以下步驟處理。

## 1. 內容定位

快速判斷：
- **類型**：長文 / 討論串 / 推文串 / 社論 / 技術分享 / 引戰帖 / 資訊搬運
- **載體背景**：Twitter / Bluesky / 知乎 / 即刻 / Telegram / 公眾號 / 論壇 / 其他
- **作者立場**（若能推斷）：社群位置、一貫立場、可信度基線
- **語氣**：客觀分析 / 情緒動員 / 信息搬運 / 觀點輸出 / 釣魚

## 2. 核心結構

- **主旨**：一篇話說清楚作者想講什麼
- **論點鏈**：主論點 → 子論點 → 支撐，標出結構層次
- **證據類型**：個人經歷 / 數據引用 / 邏輯推理 / 權威背書 / 情緒共鳴
- **隱藏前提**：作者沒說但必須成立的前提。標出骨架

## 3. 分析

只做兩個維度：

- **有價值的洞察**：超出常識的觀察、好的類比、扎實的信息增量。寫一兩句為什麼有價值
- **值得警惕的點**：邏輯跳步、稻草人、倖存者偏差、情緒操弄、信息源可疑。寫一兩句問題在哪

不做無關痛癢的「可以改進」式點評。寧可留空。

## 4. 摘要

- 一句話版（15 字內）
- 三段版（每段不超過兩行）：
  1. 事實層——說了什麼
  2. 立場層——他想讓你信什麼
  3. 判斷層——你應該如何定位這篇東西

## 5. 輸出語言

預設中文輸出，保留原文中英文術語、人名、引文、專有名詞的原文不翻譯。英文單獨發帖時才用英文輸出。
